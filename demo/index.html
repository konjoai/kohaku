the idea, in plain words

What is Kohaku?

Today's AI assistants forget everything the moment a chat ends. Kohaku gives them a memory layer that actually remembers — and forgets the right things at the right speed. It draws on how human memory works: encode an experience, link related ones, replay them when relevant, let unused ones fade.

🧠

How memories are stored

Each idea becomes a brain-like pattern — thousands of numbers that capture meaning. Similar ideas get similar patterns; unrelated ones look different.

technical: 10 000-D bipolar ±1 hypervectors
🔗

Combining memories

Merge a handful of related memories into one prototype that recalls all of them. Like remembering "what cats are like" instead of every cat you've ever met.

technical: bundle (majority-vote superposition)
📏

How related two memories are

A single number from -1 to +1 tells you how closely two memories match. Higher means more related.

technical: cosine similarity
💾

Save & reload

Memory survives across sessions. Two formats: a readable JSON for inspection, and a tightly packed .hkb binary that's ~27× smaller.

technical: bit-packed serialization
🌱

Organize itself

Many noisy memories get compressed into a few clean prototypes — the same way your brain consolidates a week of experiences during sleep.

technical: bundle-of-bundles clustering
🕰️

The forgetting curve

Memories fade over time — just like in humans. Recent things stay sharp; unused ones gently fade. You set how fast.

technical: exponential decay (Ebbinghaus)

How human memory inspired this

In 1885, German psychologist Hermann Ebbinghaus taught himself nonsense syllables, then tested how many he could recall at intervals. He discovered that forgetting follows a smooth, predictable curve — sharp at first, then leveling off. Kohaku's memory model is the same shape.

Side-by-side: Ebbinghaus vs Kohaku

Pink dashed line: Ebbinghaus's published 1885 data. Amber line: Kohaku's tunable curve.

Three principles, taken from biology

1. Recency wins. A memory from 5 minutes ago is worth more than one from last month — until you replay it.

2. Similar memories merge. Your brain doesn't store every coffee you've had — it stores "coffee" and a few exceptions.

3. Unused fades, used strengthens. Spaced repetition — the principle behind Anki and Duolingo — is just a forgetting curve, queried.

What's inside .hkb

A small header, then each memory packed into a tight bit string.

// header (32 bytes)
magic        : b"KHKU"      // "this is a kohaku file"
version      : u16           // format version
dims         : u32           // pattern size, e.g. 10 000
capacity     : u32           // max memories held
next_id      : u64
timestamp    : u64           // internal clock
num_entries  : u32

// per memory
id           : u64
timestamp    : u64           // when it was learned
label        : utf-8 string
key_bits     : 1 bit per ±1   // 8× tighter than int8
value_bits   : 1 bit per ±1

The whole library, in eight lines

Encode. Combine. Save. Organize. Forget. Done.

from kohaku import (
    EpisodicMemory, encode_text,           # learn new memories
    save, load,                            # persist to disk
    consolidate_to_memory,                 # merge similar ones
    DecayConfig, query_with_decay,         # recall + forgetting
)

mem = EpisodicMemory(capacity=1000)
key = encode_text("konjo means beautiful")
mem.store(key, key, label="amharic word")

save(mem, "memories.hkb")              # 27× smaller than JSON, same recall

cfg = DecayConfig(half_life=100.0)        # memories halve every 100 ticks
results = query_with_decay(mem, key, top_k=5, config=cfg)
